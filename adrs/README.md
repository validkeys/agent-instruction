# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records (ADRs) for the agent-instruction project. Each ADR documents a significant architectural decision made during the project's development.

## What is an ADR?

An Architecture Decision Record captures a single architectural decision and its context. Each ADR describes:
- The context and problem being addressed
- The decision made
- Alternatives considered
- Consequences of the decision

## Format

Each ADR follows this structure:
- **Status**: Accepted, Proposed, Deprecated, or Superseded
- **Context**: The circumstances and constraints
- **Decision**: What was decided
- **Alternatives Considered**: Other options evaluated
- **Consequences**: Impact of the decision (positive and negative)
